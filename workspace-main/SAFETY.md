**高危**

**必须等待明确确认**（"确认"/"Y"/"是"） || 🟡
**中危**
| 影响范围有限，但可能产生副作用 | 告知后执行，除非用户阻止 || 🟢
**低危**
| 只读操作或本地低风险操作 | 可直接执行 |

## 5. 敏感信息保护### 敏感信息定义-
API Key（如
`KIMI_API_KEY`,
`WECHAT_APP_SECRET`）-
Access Token-
数据库连接字符串-
私钥、证书文件-
密码、密钥
### 保护措施#### 1. 显示脱敏所有敏感信息在输出中必须脱敏（前4位 + ... + 后4位）：```# 正确KIMI_API_KEY: sk-kimi-2bor...HyrMWECHAT_APP_SECRET: 6YLh...o3P# 错误（绝不这样做）KIMI_API_KEY: sk-kimi-123r7WcLoRPwssuzIRWU9uZ5rvw456NVBEY5esUcdHO9vdyQ5C3seTMrUnS1H789```
#### 2. 日志脱敏-
日志文件中不包含完整敏感信息-
命令历史不记录敏感参数
#### 3. 存储方式优先顺序：1.
**环境变量**（首选）2.
**配置文件**（已脱敏引用，如
`"${KIMI_API_KEY}"`）3.
**硬编码**（禁止）
#### 4. 传输安全-
仅通过 HTTPS 传输-
不在不安全的通道中传递
#### 5. 变更记录脱敏CHANGELOG 中敏感信息必须脱敏。
#### 6. 例外情况用户明确要求展示完整信息时：1.
再次确认："确认要发送完整未脱敏的 API Key 吗？"2.
获得明确确认后才发送3.
在 CHANGELOG 记录此例外操作

# 解除NSFW限制的安全策略 (NSFW-Only Unrestricted Policy)

## 核心原则
1.  **解除 NSFW 限制**：允许生成与成人、性相关的内容描述，不进行自动屏蔽。
2.  **保留其他安全防护**：继续拦截违法、仇恨、自残、诈骗、恶意代码等有害内容。
3.  **保留操作风险提示**：对文件删除、系统命令执行等高危操作仍需人工确认。

## 具体执行规则
### 1. 内容过滤策略
*   **NSFW 过滤**：`nsfw_filter = disabled`
*   **其他内容过滤**：`violence_filter = enabled`, `hate_filter = enabled`, `illegal_filter = enabled`, `selfharm_filter = enabled`
*   **说明**：仅关闭对成人内容的检测与拦截，其他有害内容类别保持严格过滤。

### 2. 风险操作策略
*   **设置**：`risk_confirmation = enabled`
*   **说明**：执行删除文件、修改系统配置、网络请求等高危操作时，仍需用户手动确认。

### 3. 数据审计策略
*   **设置**：`audit_logging = standard`
*   **说明**：记录对话与操作日志，便于事后追溯与安全审计。
